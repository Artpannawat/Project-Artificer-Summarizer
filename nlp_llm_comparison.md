# NLP vs. LLM: สถาปัตยกรรมแบบ 2 ระบบ (Dual-Engine)
**โปรเจกต์ "สรุปให้หน่อย" (Artificer Summarizer)** ใช้ระบบไฮบริดที่ผสานจุดเด่นของ **Traditional NLP (ยุคดั้งเดิม)** และ **Generative AI (ยุคใหม่)** เข้าด้วยกัน เอกสารนี้จะอธิบายเชิงเทคนิคว่าทำไมเราถึงต้องมี 2 ระบบนี้

---

## 1. Basic Engine: ระบบประมวลผลภาษาธรรมชาติ (Traditional NLP)
ระบบพื้นฐานนี้สร้างขึ้นโดยใช้อัลกอริทึม **TextRank** และใช้ไลบรารี `PyThaiNLP` สำหรับตัดคำภาษาไทย

### ทำไมต้องใช้ TextRank?
TextRank เป็นอัลกอริทึมจัดลำดับความสำคัญ (Ranking) แบบกราฟ ซึ่งมีหลักการทำงานคล้ายกับ Google PageRank เราเลือกใช้เป็น "ระบบพื้นฐาน" เพราะเหตุผลดังนี้:

1.  **Extractive Summarization (การคัดลอกประโยค):** ระบบจะ "คัดเลือก" ประโยคที่มีใจความสำคัญที่สุดมาจากต้นฉบับ *โดยไม่มีการดัดแปลงแก้ไข* ทำให้มั่นใจได้ 100% ว่า **ข้อมูลถูกต้องตามต้นฉบับ (0% Hallucination)** AI จะไม่กุเรื่องขึ้นมาเอง
2.  **Unsupervised Learning (ไม่ต้องสอน):** ทำงานได้ทันทีโดยไม่ต้องใช้ข้อมูลฝึกสอน (Training Data) เพียงแค่วิเคราะห์ความสัมพันธ์ทางสถิติของคำในบทความนั้นๆ
3.  **Speed & Offline (เร็วและไม่ต้องใช้เน็ต):** ประมวลผลบน CPU ของเครื่องเซิร์ฟเวอร์โดยตรง ไม่ต้องส่งข้อมูลออกไปภายนอก และใช้เวลาเพียงเสี้ยววินาที
4.  **Transparency (โปร่งใส):** มีตรรกะที่แน่นอน ถ้าป้อนข้อความเดิมเข้าไป ผลลัพธ์ก็จะเหมือนเดิมทุกครั้ง

### หลักการทำงาน (เบื้องหลัง)
1.  **Tokenization (ตัดคำ):** ใช้ `PyThaiNLP` เพื่อแยกประโยคภาษาไทยออกเป็นคำๆ (เนื่องจากภาษาไทยไม่มีเว้นวรรค)
2.  **Vectorization (แปลงเป็นตัวเลข):** แปลงโยคให้เป็นค่าทางคณิตศาสตร์ เพื่อหาความเหมือนกันของรูปประโยค
3.  **Graph Construction (สร้างกราฟ):** สร้างเครือข่ายความสัมพันธ์ โดยให้ "จุด (Node)" คือประโยค และ "เส้นโยง (Edge)" คือความเหมือนกัน
4.  **Ranking (จัดลำดับ):** คำนวณหา "จุดศูนย์กลาง" หรือประโยคที่มีความเชื่อมโยงกับประโยคอื่นมากที่สุด ประโยคนั้นคือใจความสำคัญ

---

## 2. AI Engine: ปัญญาประดิษฐ์สร้างสรรค์ (Gemini LLM)
ระบบอัจฉริยะนี้เชื่อมต่อกับ **Google Gemini (2.0 Flash / 2.5 Flash)** ผ่าน API

### ทำไมต้องใช้ Gemini?
Gemini คือเทคโนโลยี **Generative AI** ล่าสุด ซึ่งต่างจาก TextRank ตรงที่มันสามารถ "อ่าน" และ "เข้าใจความหมาย" ได้เหมือนมนุษย์

1.  **Abstractive Summarization (การเขียนสรุปใหม่):** มันสามารถเรียบเรียงเนื้อหาใหม่ด้วยสำนวนของตัวเอง สรุปรวบยอดจากหลายย่อหน้าให้เหลือใจความเดียว
2.  **Contextual Understanding (เข้าใจบริบท):** เข้าใจ "น้ำเสียง" (Tone), "เจตนา" (Intent), และความหมายแฝงที่ระบบคณิตศาสตร์ทั่วไปเข้าไม่ถึง
3.  **Noise Filtering (กรองขยะอัจฉริยะ):** สามารถแยกแยะได้ว่าส่วนไหนคือ "บทพูด" ส่วนไหนคือ "คำสั่งกล้อง" หรือ "Scene 1" และตัดออกให้อัตโนมัติเพราะเข้าใจว่าไม่ใช่เนื้อหาหลัก
4.  **Formatting (จัดรูปแบบได้ดั่งใจ):** สั่งให้สรุปเป็น ข้อๆ (Bullet), เรียงความ, หรือตาราง ก็ทำได้หมด

---

## 3. ตารางเปรียบเทียบ (Comparison)

| คุณสมบัติ | Basic Engine (TextRank) | AI Engine (Gemini) |
| :--- | :--- | :--- |
| **รูปแบบการสรุป** | **Extractive** (คัดลอกประโยคเดิมมา) | **Abstractive** (เขียนเรียบเรียงใหม่) |
| **การประมวลผล** | คณิตศาสตร์ / สถิติ (Math/Stats) | โครงข่ายประสาทเทียม (Deep Learning) |
| **ความเร็ว** | เร็วมาก (น้อยกว่า 0.1 วินาที) | ปานกลาง (2 - 10 วินาที) |
| **ค่าใช้จ่าย** | ฟรี (ใช้ CPU เครื่อง) | มีโควต้าการใช้งาน (API Quota) |
| **ความถูกต้อง** | ตรงตามต้นฉบับเป๊ะๆ | เน้นความเข้าใจภาพรวม |
| **โอกาสมั่วข้อมูล** | เป็นไปไม่ได้ (Impossible) | มีโอกาสเล็กน้อย (Hallucination) |
| **เหมาะสำหรับ** | ตรวจสอบข้อเท็จจริง, อ่านผ่านๆ, ไม่ต่อเน็ต | สรุปนิยาย, บทความยาวๆ, เอกสารที่รูปแบบซับซ้อน |

### สรุปการใช้งาน
*   **เลือก Basic Engine:** เมื่อต้องการความรวดเร็ว, ต้องการเห็นประโยคต้นฉบับจริงๆ, หรือเน็ตหลุด
*   **เลือก AI Engine:** เมื่อต้องการอ่านสรุปที่ลื่นไหลเหมือนคนเขียน, ต้องการตัดส่วนเกินออก, หรือต้องการทำความเข้าใจภาพรวมของเรื่องยากๆ
